# Сравнение: наш форк vs сток MKS vs FluidNC

Контекст: плата **MKS DLC32 v2.1** (ESP32, 8 МБ flash), экран TS35 **не используется**,
основной клиент — самописное Android-приложение (`app-release.apk`, Flutter, исходников нет),
которое сейчас работает со стоковой прошивкой `V2.10_H35_2022_0621_N_ZX_001`.
Дата сравнения: июнь 2026.

## Кандидаты

| | Наш форк | Сток MKS (V2.10 → V2.30) | FluidNC |
|---|---|---|---|
| База | Grbl_ESP32 1.1 + MKS + IDLEVEL + наши фиксы | Grbl_ESP32 1.1 + MKS (closed-source сборки) | Наследник Grbl_ESP32, переписан |
| Версия | main (июнь 2026) | V2.30 = **последняя**, 2022-12-23 | **v4.0.3, апрель 2026** |

| Исходники | ✅ | ❌ только .bin | ✅ |

## Совместимость с приложением (`app-release.apk`) — ключевой вопрос

Приложение говорит на **ESP3D v2 протоколе**. Извлечено из APK (strings libapp.so) и
**проверено вживую на нашей прошивке**:

| Что делает приложение | Наш форк | Сток V2.10/V2.30 | FluidNC |
|---|---|---|---|
| `HTTP /command?commandText=...` | ✅ проверено | ✅ (родная среда) | ⚠️ есть, но диалект другой |
| Wi-Fi-провижининг `[ESP100/101/110/115]` | ✅ проверено | ✅ | ⚠️ частично переименовано |
| Телнет-порт 8080 (`[ESP131]8080`) | ✅ проверено (порт оставлен 8080) | ✅ | ⚠️ настройка есть, синтаксис другой |
| Raw-телнет: G-код, `?`-статус (`_parseGrblPosition`) | ✅ проверено | ✅ | ✅ (формат статуса совпадает) |
| `HTTP /upload print.gcode` на SD | ✅ проверено | ✅ | ⚠️ другие эндпоинты для SD |
| Запуск с SD `[ESP220]/print.gcode` | ✅ проверено | ✅ | ❌ нет ESP220 ($SD/Run) |
| Прогресс `SD:xx.xx,имя` в статусе | ✅ проверено | ✅ | ⚠️ другой формат поля SD |

Строки `[ESP740] json=yes` / `RELEASESD` в APK — мёртвые ветки кода (ни сток, ни наша
прошивка их не поддерживают — проверено по strings обоих стоковых образов; приложение
при этом со стоком работает).

**Вывод: приложение полностью совместимо с нашей прошивкой** (флоу
загрузка→запуск→прогресс прогнан на железе end-to-end). **С FluidNC оно, скорее всего,
работать НЕ будет** без доработки (нет исходников → доработка невозможна).

## Что нового в стоковых V2.20/V2.30 (vs V2.10) — стоит ли обновлять сток?

По официальному changelog MKS (TS35/Laser/Normal):

- V2.20 (2022-11): UI экрана, оптимизация передачи файлов, переключение Wi-Fi с экрана.
- V2.30 (2022-12): фиксы UI экрана, «не находился Wi-Fi», **отключён Wi-Fi автореконнект**.

Практически всё — про **экран TS35**, который не используется. Автореконнект они
вырезали (у нас — добавлен и проверен). Партиции V2.30: один app-слот 7 МБ, **без OTA**.
**Обновляться на V2.30 смысла нет.**

## Вердикт

1. **Для текущей задачи (плата + самописное приложение, без экрана) — наш форк
   оптимален**: приложение работает (доказано на железе), все стоковые баги исправлены,
   есть OTA/новый WebUI/SD+USB — ничего из этого нет ни в стоке, ни (для приложения)
   во FluidNC.
2. **Обновляться на сток V2.30 — незачем**: изменения экранные, реконнект вырезан,
   OTA нет, исходников нет.
3. **FluidNC — кандидат на будущее**, если потребуются: VFD-шпиндель по RS485,
   сложный хоуминг, конфиг без пересборки, WebUI v3, живое сообщество. Цена перехода:
   **приложение перестанет работать** (исходников нет — не доработать), настройка YAML
   заново, и наши MKS-специфичные фиксы там не нужны/не применимы. Откат простой —
   у нас есть полный бэкап стока и `release/` с нашей прошивкой.
