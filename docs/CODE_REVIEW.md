# Code review — ревизия правок форка и кода

Многоагентное адверсариальное ревью (6 дименшенов × 2 верификатора на находку +
critic полноты). Ниже — что подтвердилось, что исправлено, что отложено и что отсеяно.

Контекст, который агенты не учли (проверено вручную):
**`ENABLE_SD_CARD` ВКЛЮЧЁН.** Машинный хедер `i2s_out_xyz_mks_dlc32.h` делает `#undef ENABLE_SD_CARD`,
но он подключается из `Config.h:66` — *раньше* `#define ENABLE_SD_CARD` (строка 101), поэтому
`#undef` срабатывает вхолостую. Эмпирически: символы `set_sd_state/closeFile/readFileLine`
присутствуют в `firmware.elf`. Значит SD-путь жив, и регрессия #1/#2 достижима.

## ✅ Исправлено (коммит ревью)

1. **РЕГРЕССИЯ (наша): отклонение/выполнение строки 2-го интерфейса во время SD-печати
   обрывало печать.** `Protocol.cpp`: рейзилось через `report_status_message()`, который при
   `BusyPrinting` на любой не-Ok-ошибке делает `closeFile()` (обрыв), а на Ok — ставит
   `SD_ready_next` (двойная прокрутка SD-потока). Переписан SD-гейт: строки 2-го интерфейса
   во время печати квитируются НАПРЯМУЮ (`grbl_send "ok"`/`grbl_sendf "error:N"`), минуя
   SD-логику. Read-only запросы выполняются, движение отклоняется — печать не страдает.
2. **Авто-дизейбл степперов — тавтология `||` (моторы отключались даже в Hold).**
   `Protocol.cpp`: `if((sys.state != Cycle) || (sys.state != Hold))` всегда истинно → на паузе
   терялся удерживающий момент (риск ухода Z). Заменено на `&&`.
3. **Переполнение буфера в `sd_get_current_filename()`.** `SDCard.cpp`: `strcpy` без границ в
   `char[50/60]` вызывающих; срабатывало в т.ч. на каждый `?`-статус во время печати при
   длинном имени/пути. Сигнатура → `(char* name, size_t cap)` + `snprintf`; все 3 вызова
   (`Protocol.cpp` ×2, `Report.cpp`) передают `sizeof`.
4. **Гонка/overflow TX-буфера WebUI-сокета** (активирована нашим включением `SERIAL2SOCKET_OUT`).
   `Serial2Socket.cpp`: `write()` звался из нескольких задач и вместе с `handle_flush()` дёргал
   нереентерабельный `broadcastBIN` без синхронизации. Теперь TX-буфер под коротким `portMUX`,
   аппенд ограничен (нет записи за границу), `broadcastBIN` — вне критической секции и только из
   `clientCheckTask`.

Проверка: `pio run -e mks_dlc32_v2_1` SUCCESS (Flash 54.8%); `pio test -e native` 5/5 PASSED.
Поведение требует подтверждения на железе.

## 🟡 Отложено / задокументировано (не исправлено намеренно)

- **`notCycleOrHold()` тавтология** (`Settings.cpp:15`, всегда false). Read-only `$$/$G` поэтому
  проходят во время движения. Чинить (`||`) НЕ стали: это ровно то, что позволяет хосту опрашивать
  `$$` во время SD-печати (фича SD↔USB). Эффект бага — безвреден (read-only). Решить осознанно.
- **Неаутентифицированный доступ по сети** (`ENABLE_AUTHENTICATION` выкл). Включив WebUI-канал, мы
  сделали живым удалённое исполнение G-кода/jog/настроек без пароля для всех в зоне AP
  (`MKS_DLC`/`12345678`). Рекомендация: сменить пароль AP (в WebUI) или включить аутентификацию.
  Realtime `0x18`/FeedHold с сокета тоже доходят (семантика GRBL).
- **После USB-прошивки новой 8 МБ-таблицы переехал SPIFFS** → ассеты WebUI стираются, нужно
  заново `pio run -t uploadfs`. NVS (настройки/Wi-Fi) НЕ теряется (стабильный offset `0x9000`).
- **`readFileLine` обрезает строку >255 и трактует как EOF** (`SDCard.cpp:206`) — длинная строка
  в G-коде может молча оборвать SD-печать. Маловероятно; отдельный фикс при необходимости.
- **Язык интерфейса `=` вместо `==`** (`mks/MKS_draw_language.cpp:8`) — косметика: в прошивке нет
  CN/DE строковых таблиц, всегда English; `$40` в NVS сохраняется. Это НЕ «имя устройства»
  (то решено сменой `DEFAULT_HOSTNAME`).
- **`board_build.flash_size`** в `ini` — возможно не читается билдером espressif32; безвреден
  (значение верное), оставлен.

## ❌ Отсеяно как ложные (адверсариальная верификация)

- «SD-код выключен `#undef`-ом» — НЕВЕРНО (см. контекст выше, SD жив).
- Удаление `set_sd_state(Idle)`-recovery — безопасно (терминальные пути закрывают файл).
- Классификатор видит полную строку; гейт таргетит правильный клиент; realtime/ответы доходят до WebUI.
- mDNS-в-AP, порядок `setHostname`, `setAutoReconnect` — корректны (последний — no-op, флаг и так true).
- Гонка на RX-кольце Serial2Socket — нет (push/read в одной задаче).
- `lib_deps` Adafruit MPR121 при OFF — не баг (LDF/`--gc-sections` выкидывают, 0 байт).
- Lowercase-include в `language_ch.h` — мёртв под include-guard, сборку не ломает.
