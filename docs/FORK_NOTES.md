# FORK_NOTES — технический разбор и дорожная карта

Рабочие заметки по форку прошивки MKS DLC32 v2.1 под ЧПУ. Назначение — чтобы при
дальнейшей доработке не переоткрывать заново контекст.

## Происхождение

```
makerbase-mks/MKS-DLC32-FIRMWARE   (Grbl_ESP32 v1.1, build 2022042101, заброшен 2023-01)
        └── IDLEVEL/MKS-DLC32-FIRMWARE  (ветка MKS-DLC32-NEW-CNC, CNC-доработки)
                └── ЭТОТ ФОРК (danusha2345), ветка MKS-DLC32-NEW-CNC
```

Remotes: `origin` = форк, `upstream` = IDLEVEL, `makerbase` = оригинал (для диффов).

Почему именно IDLEVEL: из всех форков только он развивает CNC-направление (у `01power`
— лазер/CoreXY). Минусы IDLEVEL, которые правим: не собирался на Linux, бессмысленные
сообщения коммитов, ~236 МБ мусора (KiCad/G-code) в `PCB/`, код стороннего железа
включён по умолчанию.

## Что IDLEVEL добавил поверх оригинала

- **OTA** (обновление по воздуху): env `mks_dlc32_v2_1_ota`, партиция `esp32_16MiB.csv`,
  заливка через `espota` на 192.168.4.1.
- **ExternalBoard** — выносной пульт на ёмкостных кнопках `Adafruit_MPR121`, связь по
  UART1 (Zero/Home/Probe). Пины по умолчанию GPIO 4/0 (= IIC шины платы).
- **I2C** (`I2C.cpp`) — generic-обёртка, фактически мёртвый код (никто не вызывает,
  `Uart1` нигде не используется, к OTA-флагу `USE_UART_I2C_PINS` отношения не имеет —
  тот работает через `Uart0`).
- **SDCardWriter** — приём gcode в файл `/TEMP.GCODE` на SD (для загрузки по сети).
- **Telnet** переписан (`TelnetServerClient`, нормальный сетевой стрим).
- Отдельные проекты: `Joystick/` (беспроводной пульт), `GRBL_Bluetooth/` (BT-мост).
- Переход на C++17.

## Сделано в этом форке

1. **Починена сборка на Linux** — `ExternalBoard.cpp` содержал
   `#include "mks\MKS_draw_move.h"` с обратным слешом (Windows-only) → заменено на `/`.
   После этого `pio run -e mks_dlc32_v2_1` собирается (Flash ~94%).
2. **Гигиена репозитория** — удалён закоммиченный build-артефакт
   `Grbl_Esp32.ino.cpp` (содержал захардкоженные пути `C:\Users\Sergey\...`),
   добавлен `.gitignore` (`.pio/`, `.claude/`, `*.ino.cpp`).
3. **Внешний пульт → компайл-тайм переключатель.** `ExternalBoard.cpp` и `I2C.cpp`
   обёрнуты в `#ifdef ENABLE_EXTERNAL_BOARD`; мастер-флаг в `Config.h` по умолчанию
   выключен. GitNexus `impact(ExternalBoard)` подтвердил: класс импортируется только
   3 файлами (Protocol.cpp, Grbl.cpp, ExternalBoard.cpp), все под guard → risk LOW.
   Проверено сборкой в обе стороны:
   - OFF (дефолт): Flash 94.0% (1 724 042 б), MPR121 не линкуется.
   - ON (`-DENABLE_EXTERNAL_BOARD`): Flash 94.2% (1 728 954 б), пульт возвращается.
4. **OTA-партиция под реальные 8 МБ.** Создан `esp32_8MiB.csv` (2× app по 3 МБ + SPIFFS),
   в `ini` оба env переведены на него + `board_*.flash_size = 8MB`; из OTA-env убран
   `USE_UART_I2C_PINS` (уводил консольный Uart0 на GPIO0/4). Раньше OTA-env брал
   `esp32_16MiB.csv`, не влезавший в 8 МБ. Теперь USB- и OTA-сборки на одной таблице
   разделов → бинарник идентичен, плата после USB-прошивки сразу OTA-совместима.
   Проверено сборкой обоих env: Flash 54.8% (1 724 042 б из 3 МБ-слота).
5. **SD ↔ USB вместе — read-only запросы во время печати с карты.** Раньше во время
   job с SD char-гейт в `Serial.cpp` дропал любой ввод с USB/сети и слал
   `error: AnotherInterfaceBusy` на каждую строку. Теперь:
   - `Serial.cpp` всегда буферизует символы (realtime `?`/`!`/`~`/сброс и так проходили
     до гейта — мониторинг/пауза/резюм по USB во время печати работали и раньше);
   - решение принимается на уровне готовой строки в `Protocol.cpp` через чистый
     классификатор `line_safe_during_sd_job()` (`SDJobPolicy.{h,cpp}`): read-only
     запросы (`$$ $# $G $I $N $`) и пустые строки выполняются, любое движение/исполнение
     и запись настроек — отклоняются с `AnotherInterfaceBusy`.
   - Зачем: хост (LightBurn/UGS/WebUI) по USB может опрашивать состояние прямо во время
     гравировки с карты, не рискуя смешать два потока G-кода в планировщике.
   - Классификатор покрыт **нативными тестами** (`test/test_sd_job_policy/`, `pio test -e native`,
     5/5 PASSED). Поведение на железе ещё проверить.

## Заметки по плате (стоковая DLC32 v2.1 + TS35)

- Машина: `MACHINE_TYPE_DEFAULT` → `Machines/i2s_out_xyz_mks_dlc32.h`.
- `N_AXIS = 3`, `USE_BOARD_V2_0` и `USR_Z_MOTOR` определены.
- Шпиндель/лазер PWM = **GPIO32** (см. issue #45 оригинала про PWM), probe = GPIO22,
  Z-концевик = GPIO34.
- Ось Z полноценная (движение/джоггинг/probe/обнуление в UI `MKS_draw_move.cpp`).
  Z-хоминг выключен в дефолтах (DLC32 продаётся «лазерным», концевика часто нет).
- Flash платы = **8 МБ** (ESP32-WROOM-32U), см. [`BOARD_SPEC.md`](BOARD_SPEC.md). Партиция
  `esp32_8MiB.csv`: app-слот 3 МБ, прошивка ~55% — запас под фичи есть.
